# tg-ws-proxy for Android

Android-приложение, которое поднимает локальный SOCKS5-прокси для Telegram и пытается проксировать Telegram-трафик через WebSocket-маршрут.

## Для чего это нужно

Приложение запускает локальный прокси на телефоне, обычно на `127.0.0.1:1080`.

Дальше Telegram на Android подключается к этому локальному SOCKS5-прокси, а приложение:
- определяет Telegram DC;
- при необходимости патчит MTProto init;
- пытается отправить трафик через `wss://kws*.web.telegram.org/apiws`;
- при необходимости использует TCP fallback.

## Что умеет приложение

- запускать и останавливать локальный прокси;
- сохранять настройки между запусками;
- открывать ссылку для добавления прокси в Telegram;
- показывать и копировать лог;
- работать в фоне как `ForegroundService`.

## Как установить

Сейчас это исходники Android-модуля, а не готовый APK.

Чтобы собрать приложение:
1. Откройте папку `android-app/` в Android Studio.
2. Убедитесь, что используются Android SDK и JDK 17+.
3. Дождитесь `Gradle Sync`.
4. Соберите и установите приложение на устройство.

## Как пользоваться

1. Откройте приложение.
2. Проверьте адрес и порт локального прокси.
   Обычно это `127.0.0.1` и `1080`.
3. Нажмите `Start`.
4. Нажмите `Add to Telegram`.
   Если Telegram не откроется автоматически, приложение покажет ссылку и скопирует ее в буфер.
5. Подтвердите добавление прокси в Telegram.
6. Проверьте подключение в Telegram.

## Логи

Если Telegram не подключается:
1. Откройте приложение.
2. Нажмите `Show log` или `Copy log`.
3. Скопируйте лог целиком.

По логу обычно видно:
- поднялся ли локальный SOCKS5;
- подключается ли Telegram к `127.0.0.1`;
- определяется ли нужный DC;
- открывается ли WebSocket до `kws*.web.telegram.org`;
- идет ли реальный трафик через туннель.

## Если не работает

Проверьте следующее:
- сервис действительно запущен;
- в Telegram выбран именно локальный SOCKS5-прокси;
- не менялся ли порт в приложении;
- Android не остановил фоновый сервис;
- в логе есть строки `WebSocket connected`.

Если в логе есть `WebSocket connected`, но Telegram все равно не работает, нужно смотреть строки закрытия сессий и счетчики `up/down`.

## Ограничения

- это MVP, а не готовый production-релиз;
- локальная сборка из терминала в этом окружении не проверялась, потому что здесь нет Android SDK/Gradle wrapper;
- часть поведения зависит от того, как именно Telegram Android работает с локальным SOCKS5 другого приложения;
- на некоторых устройствах Android может агрессивно ограничивать фоновые сервисы.
